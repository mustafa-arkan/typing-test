#1.Depending on your needs, you can use local or session storage to store data. Both storage methods are similar to cookies but without the same privacy concerns surrounding cookies. Therefore, most modern websites prefer to use web storage methods only or combine them with cookies as a fallback.
#2.Unlike cookies, local storage and session storage do not send data to the server via HTTP headers. You should use them for client-side functionality only.
#3.Additionally, local and session storage each have a storage limit of around 5 MB per domain. They provide more storage than cookies, which only have a capacity of 4 KiB per cookie.
#4.Since local storage is persistent, it's your best bet to retain data across user visits. If you want to store site preferences or cache long-term data, local storage is appropriate. You may not want to store more sensitive data using local storage, since it is permanent.
Session storage is unique to each browser tab. If you open a new tab and navigate to the same site, you'll start a new session with its own storage. However, if you use a "duplicate tab" feature in your browser, that may reuse the same session. You can't rely too much on the specific details of a "session". Instead, focus on the core concept: session storage is temporary.

Global Scope

The variables defined outside of any function or curly brackets are known as global variables and have global scope. Global scope means that the variables can be accessed from any part of that program, any function or conditional state can access that variable.

Block Scope

Block scope is also a sub-type of local scope. The block scope can be defined as the scope of the variables inside the curly brackets {}. Now, these curly brackets can be of loops, or conditional statements, or something else. You are only allowed to refer to these variables from within the curly brackets {}.
